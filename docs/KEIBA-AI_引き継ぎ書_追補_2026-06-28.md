## 🔑 最重要トピック（2026-07-09時点）

**市場特徴量モデルが本番反映された。複勝AUCが市場と同等以上に到達。**

- 従来モデルは市場情報（オッズ・人気）を1つも使わず、複勝AUC 0.68 < 市場0.77 と**構造的に負けていた**
- 市場特徴量4個（f_popularity ほか）を追加して再学習 → **複勝AUC 0.82 ≥ 市場0.81** に逆転
- feature_importance: `f_popularity` が単独首位24.6%。予測力の大半は市場のコピー、残差が僅かに上乗せ
- ⚠ 差は小さく（+0.007, N≈130）統計的には「市場と同等〜わずか上」。フォワードで要検証
- ⚠ AUCで並んでも馬券は控除率20-25%ぶん負ける。勝つには「市場が誤る領域をAIが当てる」ことが必要
- 詳細は CLAUDE.md「2026-07-09 セッション」参照

**⚠ market_correction.py は廃止済み（存在しない）。** DB内の correction_* 列は履歴の残骸で、
init_db が自動クリアする。過去の「市場補正レイヤー」記述は無効。

## システム全体像

```
スマホアプリ（index.html）
    ↑ latest.json / stats.json
GitHub Actions（金/土/日ワークフロー）
    ↓ main checkout → 実行
scripts/friday_predict.py   ← 土曜レース予想生成
scripts/weekend.py          ← 土曜結果取得 + 日曜予想生成
scripts/sunday_results.py   ← 日曜結果取得・DB保存・ROI集計
    ↓ 予測エンジン
src/features/engine.py      ← 特徴量計算（XGB + ルールベース）
src/models/predict.py       ← softmax / キャリブレーション
src/betting/make_bets.py    ← 旧買い目生成（ルールベース）
src/betting/app_json.py     ← latest.json 生成（gumbel_bets も含む）
    ↓ 新買い目エンジン（2026-07-02〜）
src/betting/bet_optimizer.py   ← EV買い目生成（Gumbel確率ベース）
src/betting/dual_model.py      ← B2_ndcg（単勝）+ A_fukusho（複勝等）
src/betting/race_simulator.py  ← Gumbel-Max シミュレーション
src/betting/payout_estimator.py ← 推定配当計算
```

## 温度校正結果（実測値）

```
【rating_temperature.json（4-5月val + フォワード）】
モデル             T      ECE     用途
A_fukusho         0.7    0.0136   dual_modelパス（現状本番未使用）
B2_ranking_ndcg   1.0    0.0043   dual_model単勝（feat_df渡し時）
pairwise          5.0    0.0064   不使用
gumbel_rating     2.5    0.0095   ★本番のmake_bets_v2（feat_dfなし）で使用

【softmax温度（engine.py ハードコード）】
win_prob softmax  T=3.5           RL1過信解消（ECE 0.0357→0.0169）
```

> ⚠ **これらの温度は全て旧モデル（AUC0.69）のデータでフィットしたもの。**
> 2026-07-09の新モデル（AUC0.82, 市場特徴量入り）は確率分布が変わっており、
> **フォワードデータ蓄積後に softmax T / gumbel_rating T を再フィットする必要がある**。
> 今すぐ変えると二重補正リスク。CLAUDE.md「2026-07-09」の温度注記を参照。
